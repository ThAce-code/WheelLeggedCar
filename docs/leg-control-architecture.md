# 轮腿舵机控制架构规划

## 1. 背景

当前小车使用 4 个舵机组成轮腿机构，布局按当前图纸暂定为：

| 舵机编号 | 位置定义 | 备注 |
| --- | --- | --- |
| `SERVO_FL = 0` | 前左 | 图中左上 |
| `SERVO_FR = 1` | 前右 | 图中右上 |
| `SERVO_RL = 2` | 后左 | 图中左下 |
| `SERVO_RR = 3` | 后右 | 图中右下 |

如果后续实车车头方向或左右定义与该图相反，只调整映射配置，不修改控制算法。

## 2. 舵机在整车中的作用

无刷电机负责轮子的速度和驱动力，舵机负责轮腿机构的几何姿态。舵机不直接决定小车前进速度，而是决定车身相对轮子的支撑形态。

舵机主要作用如下：

1. 控制车身高度或轮腿展开程度。
2. 调整前后支撑差异，用于俯仰姿态补偿。
3. 调整左右支撑差异，用于横滚姿态补偿。
4. 为起步、刹车、越障、姿态恢复等动作提供机构姿态。
5. 在异常姿态或调试阶段提供固定安全姿态。

因此，电机控制可以理解为“跑”，舵机控制可以理解为“站姿”和“支撑”。

## 3. 控制分层

舵机控制需要避免上层控制直接操作 PWM。推荐分层如下：

```text
IMU / 视觉 / 遥控 / 轮速反馈
        |
        v
estimator / vehicle_state
        |
        v
motion_manager
        |
        +--> control_chassis
        |       `-- motor_cmd
        |
        `--> control_leg
                `-- servo_cmd
                        |
                        v
                actuator_servo
                        |
                        v
                    PWM 输出
```

各层职责：

| 层级 | 职责 | 不应该做的事 |
| --- | --- | --- |
| `sensor_*` | 采集 IMU、轮速、视觉等数据 | 不直接控制舵机 |
| `estimator` | 形成车身姿态、速度等状态快照 | 不输出 PWM |
| `motion_manager` | 选择当前动作模式和目标姿态 | 不关心具体舵机脉宽 |
| `control_leg` | 将车身姿态目标混控为四个舵机角度 | 不直接操作 PWM 寄存器 |
| `actuator_servo` | 限幅、缓动、使能、PWM 输出 | 不理解车身姿态语义 |

## 4. 控制输入与输出

`control_leg` 的输入应是车身层目标，而不是单个舵机角度：

| 输入 | 含义 |
| --- | --- |
| `height_cmd` | 车身高度或轮腿展开程度 |
| `pitch_cmd` | 前后俯仰补偿量 |
| `roll_cmd` | 左右横滚补偿量 |
| `action_offset[4]` | 特殊动作给每个轮腿的偏置 |
| `mode` | 当前腿部控制模式 |

`control_leg` 的输出是统一的 `servo_cmd_struct`，交给 `actuator_servo_set_cmd()`。

## 5. 四舵机混控模型

每个舵机只在配置中保存自己的几何属性：

| 字段 | 说明 |
| --- | --- |
| `neutral_deg` | 机械中位角 |
| `min_deg/max_deg` | 机械软限位 |
| `direction` | 舵机安装方向，取 `+1` 或 `-1` |
| `mount_x` | 前后位置，前为 `+1`，后为 `-1` |
| `mount_y` | 左右位置，左为 `+1`，右为 `-1` |

建议的基础混控：

```text
servo_angle[i] =
    neutral_deg[i]
  + direction[i] * (
        height_cmd
      + pitch_cmd * mount_x[i]
      + roll_cmd  * mount_y[i]
      + action_offset[i]
    )
```

这样可以让上层只表达“车身抬高”“车头压低”“左侧抬高”等意图，不关心 0/1/2/3 每个舵机实际应该正转还是反转。

## 6. 控制模式

初期只保留 3 个模式：

| 模式 | 行为 | 用途 |
| --- | --- | --- |
| `LEG_MODE_LOCK` | 输出固定安全角或保持中位 | 上电、IMU/单电机测试、故障态 |
| `LEG_MODE_MANUAL` | 允许手动设置四个目标角 | 标定方向、中位和机械限位 |
| `LEG_MODE_ATTITUDE` | 使用 IMU roll/pitch 做小幅补偿 | 后续整车姿态联调 |

暂不建议一开始就做复杂动作序列。先把四个舵机的方向、软限位和同步性调稳，再进入动态姿态补偿。

## 7. 安全约束

舵机输出路径必须固定为：

```text
目标角度 -> 机械软限位 -> 速度限制 -> 使能判断 -> PWM 输出
```

关键安全规则：

1. 上电默认进入 `LEG_MODE_LOCK`，不自动往复动作。
2. 未完成单舵机方向确认前，不允许四舵机同步大幅动作。
3. 每个舵机必须有独立软限位。
4. `actuator_servo` 保留缓动限制，避免目标角突变。
5. 姿态异常、IMU 失效或急停时，立即进入安全姿态或关闭舵机输出。

## 8. 初期联调顺序

1. 固定 `APP_SERVO_TEST_ENABLE=0`，保持自动舵机测试关闭。
2. 建立 `leg_config`，写入四个舵机的位置、方向、中位和软限位。
3. 用 `LEG_MODE_MANUAL` 单独测试 0、1、2、3。
4. 验证每个舵机的正方向是否符合“轮腿展开为正”的统一定义。
5. 验证四舵机中位输出。
6. 验证 `height_cmd` 同步展开/收回。
7. 验证 `pitch_cmd` 前后差动。
8. 验证 `roll_cmd` 左右差动。
9. 小幅接入 IMU roll/pitch 补偿。

## 9. 当前结论

舵机模块应作为轮腿机构的姿态执行层，不应和电机速度闭环混在一起。后续代码重点不是继续写 PWM，而是建立 `leg_config` 和 `control_leg` 的姿态混控，让 4 个舵机从“单个角度输出”升级为“车身姿态执行器”。
