# WheelLeggedCar 项目规划文档

## 1. 项目背景

本项目基于成都逐飞科技 CYT2BL3 开源库空工程开发，目标平台为 Infineon CYT2BL3 微控制器，开发环境为 IAR Embedded Workbench for ARM。当前工程已完成基础工程初始化，并已开始验证 LSM6DSV16X 六轴 IMU 的姿态角读取与 VOFA+ 可视化输出。

轮腿小车相比普通两轮车或四轮车，对姿态感知、执行器同步和控制时序要求更高。项目需要同时接入摄像头、IMU、两个无刷电机和四个舵机，其中 IMU 与电机控制属于高频闭环，摄像头属于低频感知，舵机属于中频执行机构。如果模块边界和调度时序不清晰，后续容易出现数据延迟、控制互相阻塞、执行器输出不一致等问题。

因此，本项目在功能开发前先建立清晰的软件架构和阶段计划，确保各模块解耦、时序逻辑自洽，并为后续调试、扩展和比赛策略迭代留出空间。

## 2. 项目目的

项目最终目标是完成一套可用于轮腿小车的嵌入式控制软件，实现姿态感知、视觉感知、轮腿执行和调试可视化能力。

核心目的如下：

1. 建立稳定可靠的底层硬件驱动接入能力。
2. 实现 IMU 姿态数据读取，为车体平衡与姿态保护提供输入。
3. 实现摄像头图像采集与基础路径特征提取，为循迹或目标识别提供输入。
4. 实现两个无刷电机的独立控制与速度反馈闭环。
5. 实现四个舵机的独立控制，为轮腿机构姿态调整提供执行能力。
6. 建立统一调度框架，使高频控制、中频执行、低频视觉和调试输出互不阻塞。
7. 形成可逐步验证的开发流程，每一阶段都有明确验收标准。

## 3. 功能范围

### 3.1 传感器模块

- IMU：LSM6DSV16X，优先使用芯片内置 SFLP 四元数输出，必要时保留互补滤波作为备用方案。
- 摄像头：逐飞 MT9V03X，完成初始化、帧完成检测、图像缓存读取和基础图像处理。
- 编码器或电调反馈：用于两个无刷电机速度估计，具体接口根据硬件接线确定。

### 3.2 执行器模块

- 无刷电机 2 个：实现左右电机独立控制、限幅、急停和速度闭环。
- 舵机 4 个：实现角度命令输出、软限位、缓动和中位校准。

### 3.3 控制与算法模块

- 姿态估计：整合 IMU 姿态角、角速度和轮速信息。
- 车体控制：实现速度、转向和平衡相关控制接口。
- 腿部控制：实现四舵机目标角度生成、动作约束和机构保护。
- 视觉处理：实现图像二值化、边线/中心线提取、偏差输出。
- 安全保护：实现姿态超限、通信异常、传感器失效、输出异常时的安全停机策略。

### 3.4 调试与观测模块

- VOFA+：用于姿态角、轮速、电机输出、舵机角度、视觉偏差等关键变量可视化。
- 串口日志：仅用于低频状态提示和错误信息，不进入高频控制路径。
- LED 状态：用于快速判断运行状态、故障状态和初始化状态。

## 4. 架构原则

1. 传感器模块只负责采集和发布数据，不直接控制执行器。
2. 控制模块只读取传感器快照并输出控制命令，不直接操作底层 PWM。
3. 执行器模块只接收命令、限幅并输出硬件信号。
4. 中断服务函数只做计数、置标志或清标志，不做复杂计算。
5. 所有关键数据必须带时间戳，控制逻辑只使用最新且可信的数据。
6. 高频任务优先保证确定性，低频任务不得阻塞高频闭环。
7. 每个阶段先完成可观测验证，再进入下一阶段开发。

## 5. 初步时序规划

| 任务 | 建议周期 | 说明 |
| --- | ---: | --- |
| 系统 tick | 1 ms | 由 PIT 提供统一时间基准 |
| 安全保护 | 1 ms | 优先级最高，必要时覆盖所有输出 |
| 电机输出更新 | 1 ms | 保证无刷控制响应及时 |
| 编码器/轮速更新 | 1 ms | 为速度闭环提供反馈 |
| IMU 姿态更新 | 5 ms | 约 200 Hz，用于姿态和平衡控制 |
| 姿态/车体估计 | 5 ms | 读取 IMU 与轮速快照 |
| 平衡/速度控制 | 5 ms | 高频控制闭环 |
| 舵机命令更新 | 10 ms | 约 100 Hz，输出目标角度 |
| 摄像头帧处理 | 20 ms | 对应约 50 fps 图像输入 |
| 视觉控制修正 | 20 ms | 输出路径偏差与转向修正 |
| VOFA+ 输出 | 20-50 ms | 调试观测，不参与闭环实时性 |

## 9. 主要风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| IMU FIFO 积压导致姿态延迟 | 平衡控制滞后 | 每次读取 FIFO 最新帧，必要时降低 SFLP ODR 或提高读取频率 |
| VOFA+ 输出占用串口时间 | 干扰控制周期 | 调试输出降频，禁止在高频中断中 printf |
| 摄像头图像处理耗时过长 | 阻塞高频控制 | 图像处理放低频任务，必要时只处理 ROI |
| 无刷电机响应不一致 | 车辆跑偏或姿态不稳 | 左右电机独立标定，速度闭环参数分开管理 |
| 舵机机械限位不清楚 | 损坏轮腿机构 | 先做中位和软限位标定，再允许自动动作 |
| 模块间直接调用过多 | 后期难维护 | 统一使用状态快照和命令结构体传递数据 |
| PWM 引脚与调试串口复用冲突 | 调试中断或执行器无输出 | `P00_0/P00_1` 当前优先保留给 UART0，舵机 2/3 启用前需要重新分配资源 |
| 舵机独立供电未共地 | PWM 正常但舵机不动作 | 舵机电源 GND、MCU GND、电源负极必须共地 |
