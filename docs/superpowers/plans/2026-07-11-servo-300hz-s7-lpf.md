# 300 Hz Servo S7 and LPF Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the servo actuator's constant-speed follower with a 300 Hz first-order low-pass execution loop, add shared S7 leg trajectories, and preserve direct-step and fault safety behavior.

**Architecture:** `control_leg` remains a 10 ms motion planner and produces synchronized four-servo targets. A pure `servo_motion` unit performs LPF, rate-guard, and settle calculations; `actuator_servo` runs that unit from `PIT_CH1` every 3333 us and publishes PWM-only diagnostics through an interrupt-safe snapshot. TCPWM emits 500–2500 us pulses at 300 Hz, while `LJ` uses an explicit immediate bypass.

**Tech Stack:** Embedded C99, CYT4BB/Traveo II TCPWM and PIT, IAR Embedded Workbench 9.40.1, PowerShell numeric/static tests, host GCC test harnesses.
