# 300 Hz 舵机 S7 轨迹与一阶低通控制设计

日期：2026-07-11
目标分支：`codex/leg-control-speed-assist`

## 1. 目标

参考 `E3_06_servo_control_demo`，改进当前四路轮腿舵机控制，在保持安全边界和四腿同步的前提下：

- 将舵机 PWM 载波从 50 Hz 提升到 300 Hz；
- 使用 demo 的一阶低通算法替换执行层的恒速追踪，降低目标变化引起的抖动和机械共振；
- 将快速高度轨迹从 S5 升级为 S7，使起止点速度、加速度和 jerk 均为零；
- 提高高度动作响应，同时避免直接照搬 demo 的固定 2.8 秒轨迹、固定占空比中心和两舵机假设；
- 保留直接阶跃诊断能力和故障回安全角能力。

该系统仍为开环 PWM 舵机系统。软件记录的是输出命令，不是舵机实测角度。

## 2. 当前实现与 demo 的差异

当前项目：

- `control_leg` 以 10 ms 周期生成四路角度目标；
- 常规高度模式使用速度、加速度和 jerk 约束的高度参考；
- 快速高度模式使用 S5 位置曲线；
- `actuator_servo` 使用 `step_to_target()` 按最大角速度恒速逼近目标；
- PWM 频率为 50 Hz；
- `LEG_MODE_DIRECT_STEP` 可绕过恒速追踪并立即写入目标。

参考 demo：

- PWM 和执行更新均为 300 Hz；
- 使用共享 S7 轨迹生成目标；
- 使用 `filtered += 0.05 * (target - filtered)` 一阶低通；
- 使用固定占空比中心、固定占空比步长和固定 840 tick 轨迹；
- 只同步两个启用的舵机。

本设计复用 S7 和一阶低通思想，不复制 demo 的占空比常量、固定运动时间或舵机数量假设。

## 3. 总体架构

控制分为三层：

1. `control_leg` 运动规划层，周期 10 ms：生成高度或姿态参考，并保证四腿使用同一个归一化轨迹进度。
2. `actuator_servo` 执行层，周期约 3333 us：对最新角度目标进行一阶低通和最终安全限速，然后更新四路 PWM 比较值。
3. TCPWM 硬件输出层：以 300 Hz 连续输出 0.5–2.5 ms 脉宽。

`PIT_CH0` 继续提供 1 ms 系统时基。空闲的 `PIT_CH1` 配置为 3333 us，调用新的舵机执行 tick。`app_scheduler` 不再以 10 ms 周期调用 `actuator_servo_update()`。

## 4. PWM 配置和角度映射

配置改为：

```c
#define APP_SERVO_PWM_FREQ_HZ       (300U)
#define APP_SERVO_CONTROL_PERIOD_US (1000000U / APP_SERVO_PWM_FREQ_HZ)
#define APP_SERVO_MIN_PULSE_US      (500U)
#define APP_SERVO_MID_PULSE_US      (1500U)
#define APP_SERVO_MAX_PULSE_US      (2500U)
```

角度先受全局 `0–180°` 和每个舵机 `leg_config` 范围约束，再映射到脉宽。占空比按频率直接计算：

```text
duty = pulse_us * PWM_DUTY_MAX * APP_SERVO_PWM_FREQ_HZ / 1,000,000
```

库中 `PWM_DUTY_MAX = 20000`，因此 300 Hz 下应得到：

- 0° / 500 us：3000；
- 90° / 1500 us：9000；
- 180° / 2500 us：15000。

不得复制 demo 的 `STEER_DUTY_SAFE_MIN=3500` 和 `STEER_DUTY_SAFE_MAX=5500`，因为当前项目的真实安全角范围由 `leg_config` 管理。

## 5. 执行层一阶低通

### 5.1 正常动作

每路舵机保存：

- 最新目标角 `target_angle_deg`；
- 低通状态 `filtered_angle_deg`；
- 实际写入 PWM 的命令角 `output_angle_deg`。

300 Hz tick 中执行：

```text
filtered += alpha * (target - filtered)
candidate = filtered
output = clamp_rate(output, candidate, max_speed / 300)
output = clamp_angle(output, servo_min, servo_max)
```

默认 `alpha = 0.05`。在 300 Hz 下，其等效时间常数约为 65 ms。低通成为正常跟随算法；最大角速度限制保留为安全护栏，只在大阶跃、异常命令或重定向时限制单 tick 变化。

正常速度上限继续使用 `APP_SERVO_MAX_SPEED_DPS = 90°/s`，对应每 tick 最大 0.3°。快速高度模式使用 `APP_LEG_FAST_SERVO_MAX_SPEED_DPS = 180°/s`，对应每 tick 最大 0.6°。

增加执行层收敛判据：所有启用舵机的 `abs(target_angle_deg - output_angle_deg)` 均不超过 `APP_SERVO_SETTLE_ERROR_DEG = 0.2°` 时，执行层才报告 settled。该状态必须连续保持 `APP_SERVO_SETTLE_MS = 100 ms`，以避免量化或重定向造成状态抖动。

### 5.2 初始化、使能和停用

- 初始化时目标、低通状态和输出状态均设为 90°，PWM 保持关闭；
- 首次使能从当前保存的输出状态连续启动，不从零占空比参与滤波；
- 停用时立即将对应 PWM duty 写为 0，并清除 enable；
- 再次使能前，将低通状态与当前安全输出状态对齐，避免恢复时跳变；
- NaN、越界角度或非法速度配置不得进入滤波状态。

### 5.3 直接阶跃旁路

`LEG_MODE_DIRECT_STEP` / `LJ` 保持诊断语义：

- 在短临界区内立即把目标角同步到 target、filtered 和 output 三组状态；
- 立即更新四路 PWM；
- 退出临界区后，300 Hz tick 从新的状态继续运行，不产生回跳。

临界区使用项目已有的 `interrupt_global_disable()` / `interrupt_global_enable()`，避免与 `PIT_CH1` 同时修改执行状态。

### 5.4 故障回安全角

故障路径继续把四路目标设置为各自 `safe_deg`，但不旁路低通。回安全角同时经过一阶低通、正常速度安全限幅和每舵机角度限制。PWM 只有在应用安全策略要求完全停用时才立即清零。

## 6. 主循环到 300 Hz 中断的数据交接

`control_leg` 在主循环上下文运行，`actuator_servo` tick 在 `PIT_CH1` 中断运行。四路角度、enable、速度档位和旁路标志必须作为同一帧发布，不能让中断读到半帧数据。

采用双缓冲命令帧：

- 主循环只写 inactive 帧；
- 写完所有字段后，以一个 `uint8` 索引原子切换 active 帧；
- 中断在 tick 开始时只读取一次 active 索引，并在本 tick 内使用同一帧；
- 中断不调用 `control_leg`，主循环也不执行正常 PWM 写入。

直接阶跃是唯一例外，必须通过短临界区同步执行状态和 PWM。

## 7. S7 轨迹设计

### 7.1 多项式

使用 demo 的七次平滑进度函数：

```text
p(u) = 35u^4 - 84u^5 + 70u^6 - 20u^7,  0 <= u <= 1
```

它在 `u=0` 和 `u=1` 处满足位置精确到端点，且速度、加速度和 jerk 均为零。

### 7.2 快速高度

`LEG_MODE_FAST_HEIGHT` 将现有 S5 blend 替换为 S7。运动时间继续使用 `fast_height_transition_ms = 500 ms`，起点为收到命令时的当前高度参考，终点为新目标高度。

中途收到新的快速高度命令时：

- 以当时的高度参考作为新起点；
- 以新命令作为终点；
- 重新从 `u=0` 规划；
- 不把参考值跳回上一段轨迹的起点。

`progress == 1` 只表示上层规划完成。由于一阶低通仍可能使最终 PWM 命令滞后，快速高度模式必须继续保持 `LEG_MOTION_TRANSITION`，直到执行层 settled 连续保持 100 ms，才进入 `LEG_MOTION_STABLE`。

### 7.3 离散姿态命令

`LIKREF`、`LXY` 和手动角度使用一个四舵机共享的 S7 进度：

- 起点取当前四路规划输出；
- 终点取新的四路安全角目标；
- 四路使用相同 `u`，同时起步、同时结束；
- 轨迹时间按四路最大角度差计算：

```text
duration_s = max(0.10, 2.1875 * max_abs_delta_deg / speed_limit_dps)
```

其中 2.1875 是该 S7 进度函数的归一化峰值速度。最终执行层低通和速度护栏仍可限制异常变化，但不允许破坏每路安全角范围。

离散姿态轨迹达到 `u=1` 后同样等待执行层 settled，再把腿部运动状态标记为稳定。常规高度模式原有的高度参考 settle 条件也必须与执行层 settled 同时成立，底盘过渡期限速才可解除。

## 9. 诊断与遥测

保留现有 `servo*_output_deg` 语义，表示最终写入 PWM 的开环命令角。增加或明确以下诊断状态：

- 四路原始目标角；
- 四路低通角；
- 四路最终输出角；
- 当前速度档位；
- 执行层 settled 状态和最大目标/输出角误差；
- S7 轨迹模式、归一化进度和剩余时间；
- 直接阶跃旁路是否激活。

诊断不得把输出命令描述为实际舵机角度或实际腿高。

平衡遥测使用固定 46-float VOFA 帧，字段顺序必须在固件、通用采集工具和 IK 校准工具之间保持一致：

- 0–11：时间、电机、平衡和 IMU 核心状态；
- 12–17：腿模式、高度命令、IK 有效性和输出使能；
- 18–21：四路最终输出角；
- 22–25：四路规划目标角；
- 26–29：四路一阶低通角；
- 30–32：最大目标/输出误差、settled 和 S7 归一化进度；
- 33–34：左右 IK 目标 Y；
- 35–39：高度参考、速度、IK 裕量、运动状态和故障原因；
- 40–45：前进限速、允许行驶、快速档、直接旁路、S7 轨迹模式和剩余时间。

遥测生成周期为 10 ms。46-float 帧连同 4 字节尾标记共 188 字节，在 460800 baud / 8N1 下线路发送时间约 4.08 ms，占 10 ms 周期约 40.8%。

不得在协作调度器内调用逐字节等待 `TX complete` 的阻塞发送。`telemetry_update()` 只生成一帧并发布给应用层 TX 状态机；`telemetry_service()` 由主循环每次迭代调用，通过 SCB TX FIFO 的非阻塞写接口持续推进发送。发送中的帧不可被覆盖；若上一个 10 ms 周期的帧仍未完成，则丢弃新帧并保留当前发送，避免队列积压和实时任务抖动。该实现不得修改 vendor UART 驱动。

## 10. 测试策略

实施采用测试先行。至少增加以下主机侧或静态测试：

1. 300 Hz PWM 映射：验证 500/1500/2500 us 对应 3000/9000/15000 duty。
2. 一阶低通：验证 alpha 为 0.05、阶跃响应单调、无过冲、误差按 0.95 比例衰减。
3. 速度护栏：验证正常模式每 tick 不超过 0.3°，快速模式不超过 0.6°。
4. S7 数值性质：验证端点、单调性以及端点速度、加速度和 jerk 为零。
5. 四舵机同步：不同角度差使用同一进度并同时完成。
6. 动态重定向：新命令从当前规划值开始，不回跳、不越过新目标。
7. 双缓冲交接：验证一帧包含四路角度、enable 和速度档位，ISR 不读取半帧。
8. 直接阶跃：验证 `LJ` 同步 target/filtered/output 并立即写 PWM。
9. 故障路径：验证安全角经过低通和速度护栏，安全停用仍立即 PWM=0。
10. 稳定状态门控：验证规划完成但输出误差仍大于 0.2° 时保持 `TRANSITION`，误差连续达标 100 ms 后才进入 `STABLE`。
11. 遥测契约：验证 46 个固件字段与两个 PowerShell 工具逐项同序，并覆盖校准 CSV 使用的行驶许可和前进限速字段。
12. 非阻塞发送：静态验证遥测路径不调用 `debug_send_buffer()`/`uart_write_buffer()`，数值验证 188 字节帧的线路占用不超过 10 ms 周期的 50%。
13. 回归：运行现有 IK、零位标定、腿高转换、PWM 分辨率和频率检查。

完成代码后需要构建受影响的 IAR CM7_0 工程。若当前环境不能运行 IAR，则必须明确记录静态/数值测试结果，并把 IAR 构建列为上板前必做项。

## 11. 硬件验证顺序

1. 架空或可靠支撑车辆，电机驱动保持停止。
2. 示波器检查四路 PWM：周期约 3.333 ms，90° 脉宽约 1.5 ms。
3. 执行全 90° 安全参考，确认无 fault、无启动跳变和异常发热。
4. 执行 `LIKREF`，确认保持已验证的安全参考姿态。
5. 依次执行 `LXY,5,55`、`LXY,-5,55`、`LXY,0,52`、`LXY,0,58`，一次只观察一个方向结果。
6. 执行小范围 `LH`，确认无端点冲击后再执行 `LHF`。
7. 记录原始目标、低通目标、最终输出和 fault 字段，比较响应时间与振动情况。
8. 若舵机出现持续啸叫、异常发热、丢力或脉宽异常，立即停用 PWM，并回退到 50 Hz 版本排查硬件兼容性。

## 12. 非目标

本次不修改：

- 五连杆几何参数或 IK 标定结果；
- 已验证的 `(0,55)` 安全参考定义；
- 平衡、底盘速度环或转向控制增益；
- BLDC 电机协议和输出逻辑；
- `LJ` 诊断命令的直接阶跃语义；
- PWM-only 开环硬件的能力边界。
